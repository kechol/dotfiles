# プロンプトテンプレート

各タスク分類に応じてチーム生成時に使用するプロンプトの構造。

## Research Team

```
以下のタスクについてAgent Teamで多角的に調査する。

タスク: {task_description}

{N}名のチームメイトを生成してください:
{teammate_definitions}

各チームメイトは:
- 自分の調査領域を深く調べ、発見事項をまとめる
- 他のチームメイトと発見事項を共有し、互いの分析に対してフィードバックする
- 矛盾する情報や見落としがないか互いに確認する

全員の調査が完了したら、発見事項を統合した最終レポートをまとめてください。
```

## Build Team

```
以下の実装タスクをAgent Teamで並列に進める。

タスク: {task_description}

{N}名のチームメイトを生成してください:
{teammate_definitions}

ファイル担当の分離:
{file_ownership}

各チームメイトは:
- 自分の担当範囲のみを実装する（他のチームメイトの担当ファイルは編集しない）
- 共有インターフェースの変更が必要な場合は、先に他のチームメイトに通知する
- 実装完了後にテストを実行し、結果を報告する

全員の実装が完了したら、統合テストを実行し、結果をまとめてください。
```

## Debug Team

```
以下の問題についてAgent Teamで競合仮説調査を行う。

問題: {problem_description}

{N}名のチームメイトを生成してください。各チームメイトは異なる仮説を担当する:
{hypothesis_definitions}

各チームメイトは:
- 自分の仮説を検証するための証拠を集める
- 他のチームメイトの仮説に対して反証を試みる
- 科学的討論のように、証拠に基づいて議論する

チームが1つの結論に収束するか、全ての仮説が検証し尽くされるまで議論を続けてください。
最終的にfindingsドキュメントに合意内容を記録してください。
```

## Review Team

```
以下の対象をAgent Teamで多角的にレビューする。

対象: {review_target}

{N}名のチームメイトを生成してください:
{reviewer_definitions}

各チームメイトは:
- 自分のレビュー観点に集中して問題を特定する
- 発見事項に重要度（Critical / Warning / Info）を付与する
- 具体的な修正提案を含める

全レビュー完了後、重要度別に整理された統合レビューレポートをまとめてください。
```
